API Documentation

1. GET /api/inventory

Deskripsi: Endpoint ini digunakan untuk mengambil semua data barang yang ada dalam inventaris.

URL: /api/inventory

Metode HTTP: GET

Request Parameters: Tidak ada parameter yang diperlukan.

Response:

Status Code: 200 OK

[
  {
    "id": 1,
    "kode_barang": "B001",
    "nama_barang": "Laptop",
    "stok": 10,
    "harga": 15000000
  },
  {
    "id": 2,
    "kode_barang": "B002",
    "nama_barang": "Mouse",
    "stok": 50,
    "harga": 200000
  }
]

Status Code: 500 Internal Server Error

{
  "error": "Deskripsi kesalahan"
}

2. POST /api/inventory

Deskripsi: Endpoint ini digunakan untuk menambahkan data barang baru ke dalam inventaris.

URL: /api/inventory

Metode HTTP: POST

Request Body:

Pastikan request body mencakup data berikut:

  • kode_barang (string): Kode unik untuk barang.
  • nama_barang (string): Nama barang.
  • stok (integer): Jumlah stok barang.
  • harga (integer): Harga per unit barang.

Contoh Request Body:

{
  "kode_barang": "B003",
  "nama_barang": "Keyboard",
  "stok": 30,
  "harga": 400000
}

Response:

Status Code: 201 Created

{
  "message": "Barang berhasil ditambahkan"
}

Status Code: 500 Internal Server Error

{
  "error": "Deskripsi kesalahan"
}

Error Handling

  • 400 Bad Request: Parameter yang dibutuhkan tidak lengkap atau formatnya salah.
  • 404 Not Found: Endpoint tidak ditemukan.
  • 500 Internal Server Error: Terjadi kesalahan pada server saat memproses request.

API Documentation - Inventory Management

Endpoints

GET /api/inventory

Retrieve all items in the inventory.

Method: GET

Response:

{
  "kode_barang": "string",
  "nama_barang": "string",
  "stok": number,
  "harga": number
}

POST /api/inventory

Add a new item to the inventory.

Method: POST

Request Body:

{
  "kode_barang": "string",
  "nama_barang": "string",
  "stok": number,
  "harga": number
}

Response:

{
  "message": "Barang berhasil ditambahkan"
}

GET /api/inventory/:kode_barang

Retrieve a specific item by its kode_barang.

Method: GET

Response:

{
  "kode_barang": "string",
  "nama_barang": "string",
  "stok": number,
  "harga": number
}

PUT /api/inventory/:kode_barang

Update an item's details in the inventory.

Method: PUT

Request Body:

{
  "nama_barang": "string",
  "stok": number,
  "harga": number
}

Response:

{
  "message": "Barang berhasil diperbarui"
}

DELETE /api/inventory/:kode_barang

Delete an item from the inventory by its kode_barang.

Method: DELETE

Response:

{
  "message": "Barang berhasil dihapus"
}

GET /api/barangkeluar

Retrieve all outgoing items.

Method: GET

Response:

{
  "kode_barang": "string",
  "nama_barang": "string",
  "jumlah": number,
  "harga": number
}

POST /api/barangkeluar

Add an outgoing item.

Method: POST

Request Body:

{
  "kode_barang": "string",
  "nama_barang": "string",
  "jumlah": number,
  "harga": number
}

Response:

{
  "message": "Barang berhasil ditambahkan"
}

PUT /api/edit/barangkeluar/:kode_barang

Update the quantity of an outgoing item.

Method: PUT

Request Body:

{
  "jumlah": number
}

Response:

{
  "message": "Barang berhasil diperbarui"
}

DELETE /api/hapus/barangkeluar/:kode_barang

Delete an outgoing item by its kode_barang.

Method: DELETE

Response:

{
  "message": "Barang berhasil dihapus"
}
